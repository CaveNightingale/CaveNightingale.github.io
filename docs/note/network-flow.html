Model Reduction

For non-simple digraphs, we can merge parallel edges into a single edge with the sum of the capacities and remove self-loops.

For non-anti-symmetric digraphs, we can replace (u,v),(v,u)(u, v), (v, u)(u,v),(v,u) by (u,v′),(v′,v),(v,u)(u, v'), (v', v), (v, u)(u,v′),(v′,v),(v,u) where v′v'v′ is a new vertex, setting the capacity of (u,v′),(v,v′)(u, v'), (v, v')(u,v′),(v,v′) to be the same as (u,v)(u, v)(u,v).

For multiple sources and sinks, we can add a new source s′s's′, called supersource, and sink t′t't′, called supersink, connecting s′s's′ to all sources and all sinks to t′t't′.

Ford-Fulkerson Method

Definition (Residual Network)

Resiual Network GfG_fGf​ of a network GGG with respect to a flow fff is defined as the graph with the same vertices and edges as GGG and the capacity function cf:E→R+c_f: E \to \mathbb{R}^+cf​:E→R+ defined as

cf(u,v)={c(u,v)−f(u,v)if (u,v)∈Ef(v,u)if (v,u)∈E0otherwisec_f(u, v) = \begin{cases} c(u, v) - f(u, v) & \text{if } (u, v) \in E\\ f(v, u) & \text{if } (v, u) \in E\\ 0 & \text{otherwise} \end{cases}cf​(u,v)=⎩⎨⎧​c(u,v)−f(u,v)f(v,u)0​if (u,v)∈Eif (v,u)∈Eotherwise​
Algorithm (Ford-Fulkerson Method)

Coninuously find an argumenting path in the residual network GfG_fGf​ and adjust the flow fff until no argumenting path can be found.
Theorem

After the Ford-Fulkerson Method, the flow fff is a maximum flow.
Proof

First, after the Ford-Fulkerson Method, the sink ttt should not be reachable from the source sss in the residual network GfG_fGf​. Otherwise, there is an argumenting path in the residual network GfG_fGf​.

Therefore, the vertices reachable from the source sss in the residual network GfG_fGf​ form a cut (P,P‾)(P, \overline{P})(P,P) with s∈Ps \in Ps∈P and t∈P‾t \in \overline{P}t∈P. There shouldn’t be a edge from PPP to P‾\overline{P}P in the residual network GfG_fGf​, otherwise there exists a vertex in P‾\overline{P}P reachable from the source sss, which leads to a contradiction.

Therefore, the flow fff is a maximum flow.

But this does not guarantee the termination of the algorithm. If the capacities are integers, the algorithm obviously terminates since the flow value increases and bounded. But if it is not, the algorithm may not terminate since we know there are some infinate increasing sequences in real numbers.

$'fn{dfs}(u, f, t)$ [
	if $u = t 'lor f 'le 0$ [
