Euler’s Totient Function

Definition

The Euler’s Totient Function, denoted as φ(n)\varphi(n)φ(n), is defined as the number of natural number strictly less than to nnn that are relatively prime to nnn.

Properties

Property 1

φ(1)=1\varphi(1) = 1φ(1)=1
Proof

Only 000 is less than 111 and relatively prime to 111.

Property 2

φ(n)=n−1\varphi(n) = n - 1φ(n)=n−1, if nnn is prime
Proof

It’s easy to see that all positive intergers less than nnn are relatively prime to nnn if nnn is prime.

Therefore, φ(n)=n−1\varphi(n) = n - 1φ(n)=n−1.

Theorem (Euler's theorem)

If aaa and nnn are coprime positive integers, then

aφ(n)≡1(modn)a^{\varphi(n)} \equiv 1 \pmod{n}aφ(n)≡1(modn)

Fermat’s Little Theorem

Theorem (Fermat's Little Theorem)

If ppp is a prime and aaa is a positive integer that is not divisible by ppp, then

ap−1≡1(modp)a^{p - 1} \equiv 1 \pmod{p}ap−1≡1(modp)
Proof

Since ppp is a prime, φ(p)=p−1\varphi(p) = p - 1φ(p)=p−1.

Since aaa is less than ppp and ppp is a prime, aaa must be coprime to ppp.

Apply Euler’s Theorem and get ap−1≡1(modp)a^{p - 1} \equiv 1 \pmod{p}ap−1≡1(modp).

Theorem (Wilson's Theorem)
(p−1)!≡−1(modp)  ⟺  p is prime(p - 1)! \equiv -1 \pmod{p} \iff p \text{ is prime}(p−1)!≡−1(modp)⟺p is prime

  1. p is prime  ⟹  (p−1)!≡−1(modp)p \text{ is prime} \implies (p - 1)! \equiv -1 \pmod{p}p is prime⟹(p−1)!≡−1(modp)
  2. (p−1)!≡−1(modp)  ⟹  p is prime(p - 1)! \equiv -1 \pmod{p} \implies p \text{ is prime}(p−1)!≡−1(modp)⟹p is prime

p is prime  ⟹  (p−1)!≡−1(modp)p \text{ is prime} \implies (p - 1)! \equiv -1 \pmod{p}p is prime⟹(p−1)!≡−1(modp)

When p=2p = 2p=2, (p−1)!=1≡−1(modp)(p - 1)! = 1 \equiv -1 \pmod{p}(p−1)!=1≡−1(modp), which is obviously true, so we are only going to prove the case where ppp is an odd prime.

Lemma

Proof
x2≡1(modp)x2−1≡0(modp)(x−1)(x+1)≡0(modp)\begin{gather*} x ^ 2 \equiv 1 \pmod{p} \\ x ^ 2 - 1 \equiv 0 \pmod{p} \\ (x - 1)(x + 1) \equiv 0 \pmod{p} \end{gather*}x2≡1(modp)x2−1≡0(modp)(x−1)(x+1)≡0(modp)​

Euler’s Sieve

Algorithm (Euler's Sieve)

Euler’s Sieve is an algorithm to calculate φ(x)\varphi(x)φ(x) for all x≤nx \leq nx≤n and give the list of prime numbers less than nnn in O(n)O(n)O(n) time.

$'fn[INIT-PHI](n)$ [
	$p = 'varnothing$
	$'varphi = 'varnothing$
	$'varphi(1) = 1$
	for $i = 2$ to $n$ [
		$'varphi(i) = 0$
	]	
	for $i = 2$ to $m$ [
		if $'varphi(i) = 0$ [
			$'varphi(i) = i - 1$
			$p(|p|) = i$
		]
		for $j$ in $p$ [
			if $i 'cdot j > n$ [
				break
			]
			if $i 'bmod j = 0$ [
				$'varphi(i 'cdot j) = 'varphi(i) 'cdot j$
				break
			]
			$'varphi(i 'cdot j) = 'varphi(i) 'cdot (j - 1)$
		]
	]
	return $'varphi, p$
]
Proof of Correctness

By the property of φ\varphiφ, we know that φ(x)=x−1\varphi(x) = x - 1φ(x)=x−1 if xxx is prime.

And for composite xxx, suppose its minimum divisor is ddd, then we have

φ(x)={φ(xd)⋅dxmod  d=0φ(xd)⋅(d−1)xmod  d≠0\varphi(x) = \begin{cases} \varphi\left(\dfrac{x}{d}\right) \cdot d & x \mod d = 0 \\ \varphi\left(\dfrac{x}{d}\right) \cdot (d - 1) & x \mod d \neq 0 \end{cases}φ(x)=⎩⎨⎧​φ(dx​)⋅dφ(dx​)⋅(d−1)​xmodd=0xmodd=0​

φ(1)=1\varphi(1)=1φ(1)=1 is satisfied by the algorithm at the beginning and does not change.

Then consider an x>1x > 1x>1.

If xxx is composite, then there must be a minimum prime divisor ddd, such that d≤xdd \leq \dfrac{x}{d}d≤dx​. (If this is not the case, then we can make a smaller prime divisor from xd\dfrac{x}{d}dx​, which leads to a contradiction.)

This means ddd is in the prime list ppp when we reach xd\dfrac{x}{d}dx​. And by the condition of the loop, we know that φ(x)\varphi(x)φ(x) has been calculated correctly.

If xxx is prime, then φ(x)\varphi(x)φ(x) cannot be reached by previous iterations, so when we reach xxx, φ(x)=0\varphi(x) = 0φ(x)=0 and we set φ(x)=x−1\varphi(x) = x - 1φ(x)=x−1.

Therefore, the algorithm is correct.

 ⯀
Proof of Time Complexity

By the proof of correctness, we know that φ(x)\varphi(x)φ(x) is assigned exactly once for each xxx except for the initial assignment.

And there are assignment statement in all loops, so the other parts will have at most the same order of time complexity as the assignment statement.

Therefore, the time complexity is O(n)O(n)O(n).
